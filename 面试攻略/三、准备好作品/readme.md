->3、潜在竞争类：行业相近，业务相近<br/>

2、竞品分析目的<br/>
编写竞品分析报告，可随时了解竞争对手的产品和市场动态，掌握对手的产品策略，资本背景，保持对市场的敏感，为自身制定产品决策提供参考，获得新的切入点，快速自我调整并保持优势。<br/>
3、获取信息渠道<br/>
->1、公司的季报、年报<br/>
->2、公司的官网、产品论坛<br/>
->3、艾瑞，易观国际等数据网站的行业分析报告<br/>
->4、行业媒体新闻，论坛或分析文章<br/>
->5、从对方公司，相关渠道，市场运营等部门获取对方信息<br/>
->6、作为用户，使用对方产品，进行客户或者技术咨询，获取想了解的信息<br/>
->7、寻求对方产品用户（核心用户）进行访谈<br/>
->8、百度指数，Alexa.com<br/>
4、分析维度<br/>
->1、行业分析：了解行业的整体情况，如产品市场占比，包含盈利模式以及代表性产品，目标用户<br/>
->2、盈利模式：产品的盈利模式，运营的资金投入<br/>
->3、产品技术：技术是否前沿、对于用户迅速增加的压力承受能力，稳定性，技术框架<br/>
->4、战略层：产品定位，目标用户是哪些（用户特征：地域差异，年龄差异，学历差异，收入差异）？核心需求是什么？想让用户从产品中获得什么？<br/>
->5、范围层：目标用户的相关需求有哪些？应该有哪些功能？<br/>
->6、结构层：流程结构，用户如何达到某个页面，并且在他们完成事情后能到什么地方<br/>
->7、框架层：页面布局，按钮，表格，图片和文本区域的位置<br/>
->8、表现层：产皮UI和UE<br/>
->9、数据分析：包含盈利情况、推广情况、用户群体覆盖面、市场占有率、注册用户量/装机量/转化率/用活/在线市场<br/>

->1、SWOT分析方法<br/>
S(strengths)W(weaknesses)O(opportunities)T(threats) 优势，劣势，机会，威胁，将竞品的优劣势表达出来

## 2、产品分析报告
撰写产品分析报告的两种方法：<br/>
1、层次分析法<br/>
->1、战略层：产品目的，用户需求，经营者和用户想从产品中的到什么<br/>
->2、范围层：规格功能，某个功能是否应该成为该产品的功能之一，各种功能的组合方式是什么样的<br/>
->3、结构层：流程结构，用户如何到达某个页面，并且走完流程去哪了<br/>
->4、框架层：页面布局，按钮，表格和文本区域的位置，是否达到这些元素的最大效果<br/>
->5、表现层：视觉设计，产品图片，文字的表现样式，交互形式<br/>
2、递进分析法<br/>
->1、产品的定位是什么？<br/>
&emsp;&emsp;1)、产品定位决定了目标用户群体，决定了用户的使用场景和需求<br/>
&emsp;&emsp;2)、产品的定位决定了活动运营、推广、调研的群体<br/>
&emsp;&emsp;3)、产品的定位包含公司内部对产品的战略定位，产品不同的战略定位也不同，有的产品定位为盈利，有的产品定位为培养用户基础，有的产品定位为形成体验闭环。不同的产品定位，决定产品的着力点和重心不同<br/>
&emsp;&emsp;4)、产品定位决定了产品的功能的优先级，是分析产品遇到问题时重要的决定依据<br/>
&emsp;&emsp;5)、产品定位不是一成不变的，会随着试错、市场变化、用户增加、技术发展等原因而改变<br/>

->3、用户使用的场景是什么？<br/>
&emsp;&emsp;1)、在XX时间（when），XX地点（where），周围出现了某些事物时（what），特定的用户（who）萌发了什么欲望（desire），想通过XX手段（method）来解决<br/>
->4、对应场景下的需求是什么？<br/>
&emsp;&emsp;1)、源点：企业的现状<br/>
&emsp;&emsp;2)、目的地：未来实现的效果<br/>
&emsp;&emsp;3)、驱动力：问题，场景或者情况<br/>
&emsp;&emsp;4)、阻力：成本多少？需要付出什么代价？<br/>
->5、该需求的实现步骤闭环是什么样的？每步骤下有哪些维度？<br/>
->6、产品发展前景如何，产品目前处于什么阶段？<br/>
&emsp;&emsp;对市场前景的预测主要判断下面3个因素：<br/>
&emsp;&emsp;1)、目前同类产品在市场上的状况，包括品牌，质量，价位，产品附加值<br/>
&emsp;&emsp;2)、目前同类产品市场竞争情况分析，主要评估目前的竞争水平和市场空间<br/>
&emsp;&emsp;3)、产品的性价比、市场营销手段，产品综合优势和市场同类产品的对比<br/>
&emsp;&emsp;对市场前景的预测可得结论：<br/>
&emsp;&emsp;1)、产品在目前市场竞争环境下得生存空间<br/>
&emsp;&emsp;2)、产品提升市场占有率得可能性及前景<br/>

![image](https://github.com/muyexiaogui/Product/blob/master/extension/image/波特五力模型.jpeg)

该模型以目前行业内部的竞争为核心，分析行业内部情况，分析该行业上下游（购买者，供应者）的议价能力，看是否能够对其中某些环节进行取代或者减少其利润，随后分析潜入者，如行业闭环，类似产品，需求强相关产品、使用场景强相关产品等，若该行业处于初期或者变革期，甚至可以寻找一个替代品，彻底进行颠覆。<br/>
->7、市场容量，市场变化趋势，技术基础是什么？<br/>
->8、产品的易用性，功能完整性，用户体验等怎么样？<br/>
易用性：用户在使用产品过程中，觉得方便，能够得到赏心悦目的视觉感受和良好的使用体验，标志着产品的可用性和成熟度。<br/>
易用性设计到认知心理，计算机科学，图形学，语言学和社会学（易见，易学，易操作）<br/>
&emsp;&emsp;1)、易见：<br/>
&emsp;&emsp;-》功能易被发现，让用户能够一眼看到他想要的功能 （需要统计用户的操作频率，把最常用的功能放在最显眼的位置）<br/>
&emsp;&emsp;-》可定制化GUI（图形用户）让用户自己去设计操作界面<br/>
&emsp;&emsp;-》界面设计统一整洁<br/>
&emsp;&emsp;2)、易学：<br/>
&emsp;&emsp;-》好的产品不需要操作手册<br/>
&emsp;&emsp;-》操作手册以功能点进行描述，增加截图<br/>
&emsp;&emsp;-》操作手册以功能模块进行描述，增加流程图<br/>
&emsp;&emsp;3)、易操作：<br/>
&emsp;&emsp;-》操作顺手，快捷<br/>
&emsp;&emsp;-》同一功能最好在一个界面完成，避免切换页面<br/>
&emsp;&emsp;-》提供默认的操作选项（淘宝设置默认收货地址），减少用户操作次数<br/>
&emsp;&emsp;-》操作习惯，输入账号时默认调起数字键盘，输入见证码时，默认帮助用户进行提交<br/>
->9、如果做改进方案，添加什么功能？优化哪些功能？优先级如何？<br/>
&emsp;&emsp;1)、需要优化什么：<br/>
&emsp;&emsp;-》视觉负担：用户在视觉上找不到重点，看得很累<br/>
&emsp;&emsp;&emsp;&emsp;分解布局，一个页面的重点区域在哪？放置哪些内容？<br/>
&emsp;&emsp;&emsp;&emsp;区分内容层次，页面主要内容是哪些，次要内容会不会对用户造成视觉干扰<br/>
&emsp;&emsp;&emsp;&emsp;区分视觉元素，文字，图片的布局不符合对齐，亲密性等原则<br/>
&emsp;&emsp;&emsp;&emsp;需要努力定位目标信息<br/>
&emsp;&emsp;&emsp;&emsp;需要识别阅读起点<br/>
&emsp;&emsp;&emsp;&emsp;需要经常变换视线<br/>
&emsp;&emsp;&emsp;&emsp;视觉流被干扰，打断，布局不能有效对用户进行引导<br/>
&emsp;&emsp;-》认知负担：让用户很难理解产品，思考很累<br/>
&emsp;&emsp;&emsp;&emsp;需要理解不熟悉的概念和模式，应尽量减少陌生词汇、概念等学习，降低学习认知的门槛<br/>
&emsp;&emsp;&emsp;&emsp;需要理解多字，生涩的文本内容<br/>
&emsp;&emsp;&emsp;&emsp;需要理解混乱的结构和布局<br/>
&emsp;&emsp;&emsp;&emsp;需要理解模棱两可的操作<br/>
&emsp;&emsp;&emsp;&emsp;需要理解系统的状态，行为和结果<br/>
&emsp;&emsp;-》记忆负担：让用户记住乱七八糟的东西，记忆很累<br/>
&emsp;&emsp;&emsp;&emsp;需要记住对象的各种属性（名字，位置，大小，颜色）<br/>
&emsp;&emsp;&emsp;&emsp;需要记住对象的关联<br/>
&emsp;&emsp;&emsp;&emsp;需要记住操作的命令、步骤、结果<br/>
&emsp;&emsp;&emsp;&emsp;需要记住以往的操作<br/>
&emsp;&emsp;-》物理负担：让用户做无用功，身体很累<br/>
&emsp;&emsp;&emsp;&emsp;需要长距离移动鼠标或者滑动页面<br/>
&emsp;&emsp;&emsp;&emsp;需要进行多次点击<br/>
&emsp;&emsp;&emsp;&emsp;需要多种操作组合<br/>
&emsp;&emsp;&emsp;&emsp;需要切换输入模式<br/>
&emsp;&emsp;&emsp;&emsp;需要进出不同页面，连贯的操作、相关的内容应该出现在同一个区域或者页面<br/>
&emsp;&emsp;&emsp;&emsp;需要长时间的等待，需要等待的地方应该减少，尽量短，要让用户清楚地知道自己在等待，并且给用户一个等待期<br/>
&emsp;&emsp;2)、优化目标：<br/>
&emsp;&emsp;-》基于可用性的目标<br/>
&emsp;&emsp;&emsp;&emsp;易于识别/定位/阅读<br/>
&emsp;&emsp;&emsp;&emsp;易于理解/学习/记忆<br/>
&emsp;&emsp;&emsp;&emsp;易于操作<br/>
&emsp;&emsp;&emsp;&emsp;通过有效的反馈信息提供显著的系统状态<br/>
&emsp;&emsp;&emsp;&emsp;符合用户的真实世界<br/>
&emsp;&emsp;&emsp;&emsp;用户自由控制权<br/>
&emsp;&emsp;&emsp;&emsp;一致性和标准性<br/>
&emsp;&emsp;&emsp;&emsp;预防出错<br/>
&emsp;&emsp;&emsp;&emsp;识别而不是记忆<br/>
&emsp;&emsp;&emsp;&emsp;灵活、便捷地使用<br/>
&emsp;&emsp;&emsp;&emsp;美观、精简的设计<br/>
&emsp;&emsp;&emsp;&emsp;协助用户认识、分析和改正错误<br/>
&emsp;&emsp;&emsp;&emsp;帮助和说明<br/>
&emsp;&emsp;-》基于产品的目标<br/>
&emsp;&emsp;&emsp;&emsp;流程的简化（快速登陆，减少用户的注册和输入）<br/>
&emsp;&emsp;3)、检查清单：<br/>
&emsp;&emsp;-》架构和导航<br/>
&emsp;&emsp;&emsp;&emsp;架构和导航<br/>
&emsp;&emsp;&emsp;&emsp;是否采用了用户熟悉或者容易理解的结构？<br/>
&emsp;&emsp;&emsp;&emsp;是否能识别当前在网站中的位置？<br/>
&emsp;&emsp;&emsp;&emsp;是否能清晰表达页面之间的结构？对于页面层级较多的。要让用户清晰的知道，自己在的位置，如何去其他位置或者回到当前位置；<br/>
&emsp;&emsp;&emsp;&emsp;是否快速回到首页/主要首页？<br/>
&emsp;&emsp;&emsp;&emsp;链接名称与页面名称是否相对应？<br/>
&emsp;&emsp;&emsp;&emsp;当前页面的结构和布局是否清晰？<br/>
&emsp;&emsp;-》布局和设计<br/><br/>
&emsp;&emsp;&emsp;&emsp;是否采用了用户熟悉的界面元素和控件？<br/>
&emsp;&emsp;&emsp;&emsp;界面元素和控件的文字、位置、布局、分组、大小、颜色、形状等是否合理，容易识别、一致？<br/>
&emsp;&emsp;&emsp;&emsp;界面元素/控件之间的关系是否表达正确？<br/>
&emsp;&emsp;&emsp;&emsp;主要操作/阅读区域的视线是否流畅？<br/>
&emsp;&emsp;&emsp;&emsp;其他文本（称谓，提示语，提供反馈）是否一致？<br/>
&emsp;&emsp;-》内容和可读性<br/>
&emsp;&emsp;&emsp;&emsp;文字内容的交流对象是用户吗？<br/>
&emsp;&emsp;&emsp;&emsp;语言是否简洁、易懂、礼貌？<br/>
&emsp;&emsp;&emsp;&emsp;内容表达的含义是否一致？<br/>
&emsp;&emsp;&emsp;&emsp;重要内容是否处于显著位置？<br/>
&emsp;&emsp;&emsp;&emsp;是否在需要时提供必要的信息？<br/>
&emsp;&emsp;&emsp;&emsp;是否有干扰视线和注意力的元素？页面是否重点突出，然后用户能够快速将视线落在关键区域<br/>
&emsp;&emsp;&emsp;&emsp;屏幕尺寸，浏览器，手机屏幕的适配<br/>
&emsp;&emsp;-》行为和互动<br/>
&emsp;&emsp;&emsp;&emsp;是否告知、引导用户可以做什么？<br/>
&emsp;&emsp;&emsp;&emsp;是否告知需要进行哪些步骤？让用户清晰地知道完成某项工作需要经历的步骤？<br/>
&emsp;&emsp;&emsp;&emsp;是否告知需要多少时间完成？（耗时较长时间应该显示进度）<br/>
&emsp;&emsp;&emsp;&emsp;是否告知第一步做什么？<br/>
&emsp;&emsp;&emsp;&emsp;是否告知输入/操作限制？<br/>
&emsp;&emsp;&emsp;&emsp;是否有必要的系统/用户行为反馈？<br/>
&emsp;&emsp;&emsp;&emsp;是否允许必要的撤销操作？<br/>
&emsp;&emsp;&emsp;&emsp;是否页面上多有操作都必须由用户完成？<br/>
&emsp;&emsp;&emsp;&emsp;是否已将操作步骤、点击次数减少至最少？<br/>
&emsp;&emsp;&emsp;&emsp;是否所有跳转都是必须的<br/>
