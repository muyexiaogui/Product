## 1、怎么得到真实的用户需求
获取信息-> 整理统计 ->分析总结 ->功能设计 -> 实景验证<br/>
1、获取足够全的信息
 （1）、获取信息途径
    1）、公开信息（包括新闻，大众评论（微博，知乎），相关领域网站论坛）
    2）、用户调查
    3）、用户访谈（面向高质量用户）
    4）、行业专家、同类竞品从业访谈
  （2）、获取的信息
    1）、市场规模，市场特点
    2）、竞品状况，用户使用情况
    3）、目标用户特征
    4）、用户目前满足同类需求的方式
    5）、用户期望哪些方式满足需求
2、整理数据，总结信息，得出分析
3、将需求转换为功能设计
对每一个需求点都要确定出标准点（比如用户想做一款方便的快捷支付，确定方便到什么成都，快捷方式有哪些）
4、快速验证，在用户使用过程中修正需求
